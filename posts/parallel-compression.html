Parallel Compression — กลองหนักขึ้นโดยไม่เสีย Dynamic

โดย กองบรรณาธิการ · 13 มิ.ย. 2569 · 5 นาทีในการอ่าน

ปัญหาคลาสสิกของการ compress กลองคือยิ่ง compress หนักเพื่อให้ได้ความหนักและความ consistent ก็ยิ่งทำให้กลองเสีย punch และ transient ไป kick drum ที่ควรฟัง "ตุบ" กลับฟังนุ่มและแฟบลง snare ที่ควรฟัง "แตะ" กลับกลมเกินไป Parallel Compression หรือที่รู้จักกันในชื่อ New York Compression เป็นวิธีแก้ที่ engineer ใช้มาหลายสิบปีแล้ว

หลักการพื้นฐาน

แทนที่จะ compress เสียงกลองโดยตรงจนหนักตามที่ต้องการ เทคนิคนี้แยกสัญญาณออกเป็นสองทาง ทางแรกคือเสียงแห้งต้นฉบับที่ไม่ผ่าน compressor หรือผ่านน้อยมาก ทางที่สองคือเสียงที่ compress หนักมาก ratio สูง attack เร็ว release กลาง จนเสียงกลองแฟบลงเกือบราบ จากนั้นผสมทั้งสองสัญญาณเข้าหากัน เสียงต้นฉบับรักษา transient และ punch ไว้ ส่วนเสียงที่ compress หนักเติม body และ sustain ให้ ผลที่ได้คือกลองที่ฟังทั้งหนักและยังมีชีวิต

วิธีทำใน DAW

วิธีที่ง่ายที่สุดคือสร้าง Aux track หรือ Bus ใหม่ ส่งสัญญาณกลองทั้งหมด (หรือ drum bus) ไปที่ bus นี้ด้วย send แบบ pre-fader จากนั้นใส่ compressor ใน bus ตัวนี้แล้วกด compress หนักๆ ตั้ง ratio ไว้ที่ 8:1 ขึ้นไป attack ประมาณ 1-5ms release ประมาณ 100-200ms ให้ GR meter วิ่งลง 10-15 dB จากนั้นค่อยๆ blend fader ของ bus นี้เข้ามาจนได้ระดับที่พอดี DAW สมัยใหม่หลายตัวยังมีปุ่ม Mix หรือ Blend ใน compressor plugin โดยตรง ซึ่งทำแบบเดียวกันนี้ได้ใน plugin เดียวเลย

Compressor ที่เหมาะกับเทคนิคนี้

ใช้ได้กับ compressor เกือบทุกตัว แต่ที่นิยมมากคือ VCA-style compressor เช่น SSL Bus Compressor, API 2500 หรือ plugin จำลองอย่าง Waves SSL G-Master หรือ Cytomic Glue เพราะให้ character ที่ดีเมื่อถูก push หนักๆ FET compressor อย่าง 1176 ก็ให้ผลที่น่าสนใจมาก โดยเฉพาะในโหมด All-Buttons-In ที่หลายคนเรียกว่า "Nuke" mode

ไม่ได้ใช้แค่กับกลอง

แม้ว่า Parallel Compression จะดังจากการใช้กับกลอง แต่เทคนิคนี้ใช้ได้กับ source แทบทุกอย่าง เบสที่ต้องการทั้ง low-end ที่หนักและ note ที่ชัด เสียงร้องที่ต้องการทั้งความนุ่มและพลัง หรือแม้แต่กับ full mix ใน master bus ลองดูครับ ผลที่ได้มักดีกว่าที่คาดไว้เสมอ

จุดสำคัญที่สุดของ Parallel Compression คือการ blend — ไม่ต้องกลัวว่า compressed signal จะ extreme เกินไป เพราะมันจะไม่ได้ยินเดี่ยวๆ ยิ่ง compress หนักมากเท่าไหร่ blend น้อยก็ได้ผลลัพธ์เดิม ยิ่ง compress เบาก็ blend ได้มากขึ้น ทดลองหาจุดสมดุลระหว่างสองตัวแปรนี้ และจะเริ่มเข้าใจว่าทำไมเทคนิคนี้ถึงอยู่มาได้จนทุกวันนี้
